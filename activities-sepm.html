Unit 3: Software Development Life Cycles (SDLC)

  1. Identify and apply appropriate software engineering and project management methodologies, tools and techniques for the development of solutions to real-world problems.
  2. Design, develop and evaluate management systems to produce solutions that handle constraints and uncertainty, contextualising BDD concepts and secure coding principles & practices in SDLC.

RESPONSE: Software Engineering Methodologies

Methodologies Description Use Case
Waterfall Model Is a linear and sequential approach. It divides the process into distinct phases, including requirements analysis, system design, implementation, integration, testing, deployment and maintenance. Best suited for projects with well-defined requirements and little or no expected change.
Agile Model An iterative and incremental approach emphasizing flexibility, customer collaboration, and quick delivery. Includes frameworks such as Scrum and XP (Extreme Programming). Best suited for projects with rapidly changing requirements.
DevOps Model Integrates software development (Dev) and IT operations (Ops) to improve collaboration and automate processes. Components include Continuous Integration (CI), Continuous Deployment (CD), Infrasture as Code (IaC). Best suited for environments requiring continuous delivery and integration.
Spiral Model Combines the repeated testing of prototypes using the Waterfall Model's structure, emphasizing risk analysis and repeated cycles of planning, assessment, engineering and evaluation, each refining software. Best Suited for large, complex, and high-risk projects.
V-Model (Verification and Validation Model) Is an extension of the Waterfall Model that emphasizes verification and validation at each development stage. Best suited for projects where testing is critical, like safety-critical systems.

RESPONSE: Project Management Methodologies

Methodologies Description Use Case
PMBOK (Project Management Body of Knowledge) A comprehensive set of guidelines and best practices defined by the Project Management Institute (PMI) with knowledge areas such as Scope, Time, Cost, Quality, Resource, Communication, Risk Procurement and Stakeholder Management. Used in various industries for structured project management.
PRINCE2 (Projects In Controlled Environments A process-driven project management methodology that provides a clearly defined framework using principles for Business Justification, Learning from Experience or Managing by Stages. Best suited of commonly used in government projects, particularly in the UK and Europe.
Scrum An Agile Framework focusing on repeated progress through cycles designed to help teams work together more effectively by breaking down projects into small pieces allowing for continuous feedback. Suited for dynamic projects with frequent changes using tools like Product Backlog, Sprint Backlog, and Increment tracked in Trello, monday.com or ClickUp.
Lean Focuses on maximising value by eliminating waste and optimising processes. It uses principles such as Value, Value Stream, Flow, Pull and Perfection. Suited for projects with a focus on efficiency and resource optimisation.

RESPONSE: Tools for SEPMs

Tool Purpose Example
Version Control Systems Manage changes to source code, track revisions and facilitate collaborations. Git, Subversion (SVN).
Integrated Development Environment (IDEs) Provide a comprehensive environment for coding, debugging and testing. Visual Studio, Eclipse, IntelliJ IDEA.
Continuous Integration/Continous Deployment (CI/CD) Automate the building, testing and deployment of applications. Jenkins, Travis CI, CircleCI.
Project ManagementTools Manage tasks, track progress, collaborate with team members and maintain project timelines. Jira, Trello, Microsoft Project or Asana.
Collaboraton Tools Facilitate communication, document sharing and team collaboration. Slack, Microsoft Teams, Confluence
Testing Tools Automate and manage the software testing process. Selenium, JUnit, TestRail
Configuration Management Tools Automate server configuration, deployment and management. Ansible, Puppet or Chef
Risk Management Tools Identify, analyze and mitigate project risks. RiskyProject, @RISK '
