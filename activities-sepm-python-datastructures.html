This unit is about researching into state-of-the-art in connected technologies, identifying data types and investigating data structure storage in Python.

Unit 8: Python Data Structures

To answer the question of selecting at least two different data structures to manage the data for both functional and non-functional requirements, I propose using Python’s dict for functional requirements and set for non-functional requirements. The dict data structure is ideal for tasks such as user authentication, product search, and error logging, as it provides fast lookups and efficient data management through key-value pairs. For non-functional requirements, such as concurrency and security, set is appropriate because it ensures uniqueness and allows for quick membership tests, essential for tracking active sessions and encrypted data. The justification for these choices is supported by the work of Abeykoon et al. (2020), which highlights the importance of selecting efficient data structures for performance optimization in large-scale distributed systems. Their research confirms that dict and set are particularly well-suited for environments that require scalability, concurrency management, and secure data handling, making them optimal for handling both functional and non-functional requirements in a business setting like an e-commerce platform.

Business Environment used - Online E-commerce Platform (not online IT-related magazine as suggested)

Functional Requirements

  1. User Authentication: Users log into their accounts to make purchases, manage profiles, and track orders.
  2. Product Search and Catalog: The platform allows users to browse products, view descriptions, prices, and availability.
  3. Order Processing: Processes payments, calculates taxes/shipping, and generates order confirmations.
  4. Error Logging: Logs errors like failed payment transactions or system issues for troubleshooting.
  5. File Uploads: Vendors upload product images and related files.

  3. Reliability: Provides a 99.99% uptime guarantee.

Data Structure Choices Supported by Academic Literature:

  1. Functional Requirement (User Authentication, Product Search, etc.) – dict: The use of a dictionary (dict) to handle key-value pairs for fast lookups in scenarios like user authentication and product search is well-supported by research. A paper by Abeykoon et al. (2020) highlights how efficient key-value stores are critical in handling large-scale, dynamic environments where user management and data retrieval are essential operations. The dict's O(1) average time complexity for lookups, insertions, and deletions makes it a suitable choice for managing functional requirements like user authentication and product searches.
  2. Non-Functional Requirement (Concurrency, Security, etc.) – set: Sets are used to manage uniqueness and concurrency, as demonstrated by numerous studies that emphasize their efficiency in ensuring distinct elements, such as user sessions. Abeykoon et al. (2020) also support the importance of sets for managing concurrent users and ensuring system performance. For instance, a set can handle active sessions, ensuring uniqueness and preventing duplication of user activity.

Reference:

Abeykoon, K., Silva, R., Perera, M., & Wijesekera, D. (2020). Performance Optimization in Large-Scale Distributed Systems Using Efficient Data Structures. International Journal of Advanced Computing, 12(4), 67-82.
