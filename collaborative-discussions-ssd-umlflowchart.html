Contribution by Zukiswa Tuso

Developing secure software, and addressing coding weaknesses is crucial to preventing vulnerabilities, especially those listed in the OWASP Top Ten (OWASP Foundation, 2017). One notable vulnerability is the "Broken Access Control," which can expose sensitive data or allow unauthorised actions, leading to severe security issues (OWASP Foundation, 2017).

Understanding and mitigating these weaknesses and being able to visualise them through UML diagrams provides a structured approach. The Sequence Diagram therefore demonstrates the flow of interactions where the vulnerabilities may occur.

Why I Chose the Sequence Diagram

Sequence Diagram - is a type of Interaction Diagram that focuses on the message exchange and sequence of actions between objects or components.

I chose this diagram because it effectively represents how different elements, such as users, web servers, and access control mechanisms, interact over time. Sequence Diagrams offer a straightforward way to illustrate step-by-step processes, showing how access control might be breached in scenarios involving improper authorization or path traversal.

For instance, by modelling a user request to access a restricted file, we can visualize whether permission checks are adequately enforced at each stage, identifying potential gaps in the access control sequence.


This sequence demonstrates a scenario where the system fails to correctly limit file path input, allowing unauthorized access through directory traversal (../). By illustrating this flow, the sequence diagram helps highlight areas needing strict input validation and consistent authorization checks to prevent access breaches.
