Unit 2 Seminar: Scrum Security Review

Question 1:

Create a 2-column multi-line table. In the left-hand column, include the software development stages of the Scrum agile life cycle approach to project management. In the right-hand column, describe the processes which you recommend are applied at each stage to ensure that secure software is produced at the end of the development. To support the preparation of your response, you can refer to the following literature:

Sharma, A. & Bawa, R. K. (2020) Identification and Integration of Security Activities for Secure Agile Development. International Journal of Information Technology.

Scrum Development Stages Recommended Security Processes
Pre-requirement (Initial Planning) Initiate initial security education for all stakeholders to raise awareness of secure practices and define initial security requirements. This step ensures the project team understands security considerations essential to the project’s success​.
Requirement (Requirement Gathering and Analysis) Implement role matrix to establish user roles and permissions, agree on definitions for security terms, and ensure all security requirements are documented. Additionally, identify resources and trust boundaries within the system to secure architecture from the start.
Design (System and Architecture Design) Use secure design principles to enforce best practices, assumption documentation for tracking security assumptions, and conduct risk analyses to prioritize potential threats. Also, utilize quality gates to set checkpoints for security validation as the design progresses.
Daily Stand-ups (Daily Scrum Meetings) Utilize daily stand-ups to monitor security tasks, address security concerns promptly, and align the team on any identified vulnerabilities or security-related issues that arise. Regular updates keep security considerations integrated within daily work.
Implementation (Coding and Development) Apply coding rules to establish safe coding standards, use static code analysis to detect vulnerabilities early, and leverage security tools to assist in maintaining security during development. Implement automated acceptance and unit tests to verify code resilience against attacks​.
Testing (Quality Assurance and Security Testing) Perform dynamic analysis and security testing to identify real-time vulnerabilities, supplemented by vulnerability and penetration testing for critical assessment under simulated attack conditions
Retrospective(Review and Reflection) During retrospectives, review any security incidents or vulnerabilities encountered, assess the effectiveness of security processes, and identify improvements for future sprints. This reflection ensures continuous security improvement
Release (Deployment and Operational Readiness) Finalize security with operational planning and readiness to ensure proper documentation, code signing to validate integrity, and prepare incident response planning to provide action guidelines in case of a security breach. Conduct a final security review to confirm all security standards are met before release.
Maintenance (Post-Release Support) Implement Adaptive Monitoring to observe security in real-world use, update Risk Metrics based on new threats, and ensure Repository Improvement by updating security elements based on emerging issues. Regular Vulnerability Management is crucial post-release

This structure integrates practices like Microsoft’s Secure Development Lifecycle (SDL) and CLASP, ensuring that security is interwoven with agile processes without compromising speed and flexibility.


Question 2:

Some say that people are the biggest risk of cyber security. Select five terms from ISO/IEC Standard 27000 Section 3 Terms and Definitions and write a 300-word blog post on how people can be managed to overcome cyber security attacks from the inside.

Managing People to Overcome Insider Cybersecurity Threats

Cybersecurity risks often emerge from within organizations due to human actions or errors, whether intentional or accidental. The five key terms identified from ISO/IEC 27000's Section 3 are; access control, authentication, confidentiality, risk management, and vulnerability which provide a framework for understanding and managing these risks (ISO/IEC, 2018).

  1. Access control (ISO/IEC, 2018, 3.1)

    2. Ensures that access to sensitive assets is restricted based on business needs and security requirements. I order to mitigate insider threats, organizations must implement stringent access control policies, limiting data access strictly to authorised personnel. Regular audits can further ensure adherence to these controls, helping maintain organizational security (ISO/IEC, 2018, 3.3).

  2. Authentication (ISO/IEC, 2018, 3.5)

    3. This is the process of verifying an individual’s claimed identity, forming a critical barrier against unauthorised access. Multi-factor authentication (MFA) reinforces this measure by requiring additional verification steps, making unauthorised access more challenging and helping prevent internal breaches.

  3. Confidentiality (ISO/IEC, 2018, 3.10)

    4. This involves safeguarding information from disclosure to unauthorised parties. Organizations should foster a culture of confidentiality by training employees on data sensitivity and the importance of secure information handling practices. Employees should be aware of potential consequences if confidentiality is compromised, thereby ensuring a safer information environment.

  4. Risk management (ISO/IEC, 2018, 3.69)

    5. The process coordinates organizational efforts to identify, analyse, and control risks. This means employees are integral to this process; through training, they become aware of common cyber threats and internal policies, enabling them to recognise and respond appropriately to potential incidents.

  5. Vulnerability (ISO/IEC, 2018, 3.77)

    6. Were weaknesses in assets or controls may present exploitable opportunities for threats. By promoting a proactive approach to identifying vulnerabilities, organizations can engage employees in early detection and reporting. This collaborative vigilance helps to address weaknesses before external actors can exploit them.

Managing people effectively through awareness, access limitations, and clear confidentiality practices can significantly reduce cybersecurity risks from within. An informed and accountable workforce enhances an organization’s resilience against both internal and external threats.

Reference:

ISO/IEC (2018) ISO/IEC 27000:2018 Information technology — Security techniques — Information security management systems — Overview and vocabulary. Available at: https://www.iso.org/obp/ui/en/#iso:std:iso-iec:27000:ed-5:v1:en (Accessed: 6 November 2024)
